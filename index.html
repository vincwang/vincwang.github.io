Caseta Smart Away



This is an iOS project that I was involved in during my time at Lutron Electronics, working on a Smart Home control system called Caseta Wireless. This feature will allow users to set an "Smart Away" mode for the house, that will randomly change the lights in the house at night time to make a home appeared "lived-in" while it's unoccupied.

My main responsibilities include co-designing the feature spec and implementing the functionalities on iOS. The App helps users set up the feature by selecting lights, activation mode (Manual/Automatic) and etc. The App communicates with the system through a cloud service as many other iOT products.

I feel very happy to have worked on this feature with the team, because it's a feature that is directly used by customers. And truely makes me feel proud when customers are using products that I have worked on!

Technology used: Objective-C, iOS, LEAP, Nest Home/Away Assist, GeoFencing, iOT

Expression Evaluation

This project brought my understanding in Data Structures to a whole new level. It was a one-semester long project that was broken into 7 different phrases. This application recursively tokenized input expressions like (5+7*(10/2)) into numbers and operators to track the structures, which would be converted into a linked list.

One phrase of the project was to use post-fix calculation on the linked list. Another phrase was to place the numbers and operators into a expression tree, then go through the tree to calculate the result. It could also do variable calculation retrieving data from a variable tree. For instance, if user define x=10, then (x+5) will return 15. In addition, this application also supported function definition such as gcf(a,b) and sqr(a).

To look back, I am very glad that I did this project. It was very crucial in enhancing my understanding in Data Structures. Although, I struggled through every phrase of it.

Technology used: C++, Linked List, Stacks and Queues, Binary Search Trees, Mutual Recursion

CRM Portal Integration

This was a project I did for my internship at Netsolace, which provided software support to Edible Arrangements. I was given the task to integrate their CRM portal with LogMeIn Rescue Remote Connection, so that the technicians would be able to use LogMeIn Rescue functionality on their own CRM portal.

The API used Http Web Request to verify the credentials of the technician. The communication method used in my project was HTTP GET. This protocol created a query string of the name and value pairs and then appended the query string to the URL of the script on the server that handled the request. If the credentials were valid, the technician would be able to log in his account and create pin codes for customers without getting on LogMeIn Rescue's website

Technology used: ASP.NET, Visual Basic, HttpWebRequest

Recommendation System

Lutron's Smart Home product Caseta Wireless supports "scene", which allows user to define a scene by selecting a group of devices and their levels (eg: turn on Kitchen light and open bedroom shade). Scenes can be activated manually on the App or automatically through time clock schedule.

This project's goal is to monitor users' light usage activities, and find trends to suggest scheduled scenes. For example, if a user turns on bedroom light and kitchen light around 7 AM everyday, the program should suggest them to set a scheduled scene to automate this event.

On a high level view, this project can be broken down to three parts: Collect data, Analyze to find trends, Present the scene to user. It's a co-op project meant for POC purpose, therefore I have been involved from end to end of this project. The program captures the live event data from the cloud, and analyze them when it has a stash of events. Then finally it needs to send the scene to users via the App to view and edit it.

Throughout the development of this feature, there were times that I felt frustrated because it was kind like "shooting in the dark" for me to come up with algorithm to find trends in data without any data science background. But there are also times I felt very motivated and excited when I see reasonable outputs from the program and positive feedbacks from testers.

Note: This feature has only been tested on internal users at Lutron, and it's not meant for Production cutomers yet. Production Customers light usage was never monitored during the development of this feature.

Technology used: Java, Spring, MySql, Objective-C

Sales Data Analysis App

Finally did I finish my Sales Analysis Map application! It's the most robust web application that I have ever done for Edible Arrangements. This mapping application will take live store sales data from Edible Arrangements' database, and populate the data into the map. The goal is to have more visualize sales data for the company to view. Magician Agent

Magician Agent was a CRM software built with Derby database and Java. The programming design was Object Oriented. The UI was created using NetBeans GUI generator.

This software could manage bookings among customers, magicians, waitlist and holidays using relational database. It could add and delete magicians and holidays. It could also assign customers to free magicians. If no magician was free, it would waitlist the customer and assign to a magician when one became free.

Technology used: Java, Apache Derby Database

Customer Registration

This was my very first project with ASP.NET and Microsoft SQL Server. When I came to my job, I knew nothing about ASP.NET programming, which was what my company primarily used for their websites. Then I bought a .NET book for about $30 and dived into it for two weeks. My manager then told me to practice by building a customer registration dashboard.
